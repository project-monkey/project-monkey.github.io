一 基本概念

1.正例

属于某个类别的实例

2.负例

不属于某个类别的实例

3.真假

预测工作的正确与否

4.组合情况

  • TP（真正例）：实际为正样本，且被分类器预测为正样本。
  • FP（假正例）：实际为负样本，但被分类器预测为正样本。 把负样本错误的预测成了正样本，所以是假正例
  • FN（假负例）：实际为正样本，但被分类器预测为负样本。

二 重要变量

  • Precision（精确率）：预测为正样本的样本中，实际为正样本的比例。

    Precision = TP / (TP + FP)

    其中，TP表示真正例（True Positive），FP表示假正例（False Positive）。

  • Recall（召回率）：实际为正样本的样本中，被预测为正样本的比例。

    Recall = TP / (TP + FN)

    其中，TP表示真正例（True Positive），FN表示假负例（False Negative）。

  • F1-score：综合考虑Precision和Recall的指标，它是Precision和Recall的调和平均值。

    F1-score = 2 * (Precision * Recall) / (Precision + Recall)

    它的取值范围在[0,1]之间，越接近1表示分类器性能越好。

三 性能分析

  1. Accuracy（准确率）：分类器正确分类的样本数与总样本数之比。Accuracy = (TP+TN)/(TP+FP+TN+FN)。其中TP（True Positive）表示真正例数，TN（True Negative）表示真反例数，FP（False Positive）表示假正例数，FN（False Negative）表示假反例数。
  2. Macro Average（宏平均）：计算每个类别的性能指标的平均值，而不考虑类别的样本数。宏平均适用于各个类别样本数相同时。例如，宏平均精确度（Precision）是将各个类别精确度之和除以类别数。即Precision = (P1+P2+…+Pn)/n，其中P1、P2、…、Pn分别为各个类别的精确度。
  3. Weighted Average（加权平均）：计算每个类别的性能指标的加权平均值，权重为每个类别的样本数占总样本数的比例。加权平均适用于各个类别样本数不同时。例如，加权平均精确度（Precision）是将各个类别精确度乘以对应类别样本数之和再除以总样本数。即Precision = (P1N1+P2N2+…+Pn*Nn)/(N1+N2+…+Nn)，其中P1、P2、…、Pn分别为各个类别的精确度，N1、N2、…、Nn分别为各个类别的样本数。

2.命名

PEP 8建议采用不同的方式来给Python代码中的各个部分命名，这样在阅读代码时，就可以根据这些名称看出它们在Python语言中的角色。

  • 函数、变量及属性用小写字母来拼写，各单词之间用下划线相连，例如：lowercase_underscore

  • 受保护的实例属性，用一个下划线开头，例如：

  • _leading_underscore

  • 私有的实例属性，用两个下划线开头，例如：__double_leading_underscore

  • 类（包括异常）命名时，每个单词的首字母均大写，例如：CapitalizedWord

  • 模块级别的常量，所有字母都大写，各单词之间用下划线相连，例如：ALL_CAPS

  • 类中的实例方法，应该把第一个参数命名为self，用来表示该对象本身

  • 类方法的第一个参数，应该命名为cls，用来表示这个类本身

单下划线和双下划线

python中通常用下划线作为前缀来表示属性的访问权限
具体来说，有两种下划线类型，单下划线 _ 和双下划线 __

  • 单下划线表示收保护的属性，可以被访问，但是不被认为是公共的API
  • 双下划线表示为”私有属性“，他们只能在累的内部被访问，无法从外部直接访问，尝试从外部时，会返回AttributeError异常

以下为GPT给出的答案
在Python中，以单个下划线开头的参数（如_variable）通常被视为“内部使用”的参数或变量。这种约定通常用于指示变量只能在模块、类或函数的内部使用，不应被外部代码使用。

具体来说，以单个下划线开头的变量或参数通常具有以下含义：

  • _variable：这是一种常见的惯例，用于表示变量仅在其所在的模块或类的内部使用，不应被外部代码访问。但是，这只是一种惯例，实际上，这些变量仍然可以被外部代码访问和修改，只是它们的名称可能会被视为“非公开”的。

  • __variable：以双下划线开头的变量或参数是Python中的私有变量或参数。这意味着它们只能被所在的类内部访问，不能被子类或外部代码访问。如果在子类中定义了一个同名变量或参数，它不会覆盖父类中的私有变量或参数，而是会被重命名为_ClassName__variable的形式。

  • __variable__：以双下划线开头和结尾的变量或参数是Python中的特殊变量或参数。这些变量通常由Python解释器使用，例如__name__表示当前模块的名称，__doc__表示文档字符串，__init__表示类的构造函数等。

  • variable_：表示该变量是一个临时变量或私有变量

需要注意的是，使用下划线开头的变量名或参数名并不会改变它们的作用域或行为，只是一种命名约定。因此，应该避免在代码中使用双下划线开头和结尾的变量名或参数名，除非你确实需要定义一个私有变量或参数。

1.M1、M2、M3

在经济学中，M1、M2和M3都是货币供应量的不同定义，它们通常用于衡量一个国家货币总量的不同部分。以下是它们的定义：

  • M1：M1货币供应量是指一个国家流通中的所有货币形式的总和，包括现金和非利息支票存款。它通常被认为是最流动的货币供应量定义，因为它包括最直接可用于支付的货币。

  • M2：M2货币供应量包括M1，以及其他形式的存款，例如储蓄存款、货币市场基金、短期定期存款和其他较少流动的存款。因为M2包括更广泛的存款类型，所以它通常被认为比M1更宽泛。

  • M3：M3货币供应量包括M2，以及较长期的定期存款和大额存款。这些存款通常需要一定的通知或等待期才能取出，因此M3货币供应量比M1和M2更加宽泛。

市场衡量流动性时，通常使用M2，因为它包括了更广泛的货币形式，比M1更全面的反应了一个国家的货币流动性现状

2.CPI、PPI

  • CPI：消费者物价指数是一种测量消费者购买一定数量和类型的商品和服务所需支付的价格变化的指数。CPI通常用于衡量物价水平和通货膨胀率，以帮助政策制定者评估经济政策的效果。它的基础是一篮子商品和服务，例如食品、住房、医疗保健、交通、教育等。
  • PPI：生产者物价指数是一种测量生产者购买的原材料、劳动力和其他生产成本的价格变化的指数。PPI通常被认为是一个提前指标，因为生产者的成本增加通常会反映在最终产品的价格上，进而对CPI产生影响。PPI的基础是各种生产原材料、燃料和劳动力等。
