一、jupyter的常用快捷键

以下是一些常用的 Jupyter Notebook 快捷键及其作用：

  • Enter：进入编辑模式，可以编辑单元格中的内容
  • Esc：退出编辑模式，进入命令模式
  • Shift + Enter：运行本单元格并选中下个单元格，如果下个单元格不存在则新建一个单元格
  • Ctrl + Enter：运行本单元格但不选中下个单元格
  • Alt + Enter：运行本单元格并在下方新建一个单元格
  • Y：将单元格转换为代码单元格
  • M：将单元格转换为 Markdown 单元格
  • A：在当前单元格上方插入一个新单元格
  • B：在当前单元格下方插入一个新单元格
  • X：剪切当前单元格
  • C：复制当前单元格
  • V：在当前单元格下方粘贴
  • D,D：删除当前单元格
  • Z：撤销上一个操作
  • Shift + Up/Down：选中多个单元格
  • Shift + M：合并多个单元格
  • Shift + L：显示/隐藏代码行数
  • Shift + Tab：查看函数/方法的参数说明

在命令模式下输入 H，可以查看更详细的快捷键说明。

model_selection.StratifiedKFold

1.Parameters

  • n_splitint, default=5
  • shuffle= False
  • random_state=None

2.主要的作用

StratifiedKFold的作用是帮助解决数据集中类别不平衡的问题，即某些类别的样本数量比其他类别的样本数量更多或更少的情况。如果使用普通的交叉验证方法，可能会导致某些子集中某个类别的样本数量非常少，从而影响模型性能的评估。而使用StratifiedKFold可以确保每个子集中每个类别的样本数量都尽可能相似，从而更准确地评估模型的性能。

一 基本概念

1.正例

属于某个类别的实例

2.负例

不属于某个类别的实例

3.真假

预测工作的正确与否

4.组合情况

  • TP（真正例）：实际为正样本，且被分类器预测为正样本。
  • FP（假正例）：实际为负样本，但被分类器预测为正样本。 把负样本错误的预测成了正样本，所以是假正例
  • FN（假负例）：实际为正样本，但被分类器预测为负样本。

二 重要变量

  • Precision（精确率）：预测为正样本的样本中，实际为正样本的比例。

    Precision = TP / (TP + FP)

    其中，TP表示真正例（True Positive），FP表示假正例（False Positive）。

  • Recall（召回率）：实际为正样本的样本中，被预测为正样本的比例。

    Recall = TP / (TP + FN)

    其中，TP表示真正例（True Positive），FN表示假负例（False Negative）。

  • F1-score：综合考虑Precision和Recall的指标，它是Precision和Recall的调和平均值。

    F1-score = 2 * (Precision * Recall) / (Precision + Recall)

    它的取值范围在[0,1]之间，越接近1表示分类器性能越好。

三 性能分析

  1. Accuracy（准确率）：分类器正确分类的样本数与总样本数之比。Accuracy = (TP+TN)/(TP+FP+TN+FN)。其中TP（True Positive）表示真正例数，TN（True Negative）表示真反例数，FP（False Positive）表示假正例数，FN（False Negative）表示假反例数。
  2. Macro Average（宏平均）：计算每个类别的性能指标的平均值，而不考虑类别的样本数。宏平均适用于各个类别样本数相同时。例如，宏平均精确度（Precision）是将各个类别精确度之和除以类别数。即Precision = (P1+P2+…+Pn)/n，其中P1、P2、…、Pn分别为各个类别的精确度。
  3. Weighted Average（加权平均）：计算每个类别的性能指标的加权平均值，权重为每个类别的样本数占总样本数的比例。加权平均适用于各个类别样本数不同时。例如，加权平均精确度（Precision）是将各个类别精确度乘以对应类别样本数之和再除以总样本数。即Precision = (P1N1+P2N2+…+Pn*Nn)/(N1+N2+…+Nn)，其中P1、P2、…、Pn分别为各个类别的精确度，N1、N2、…、Nn分别为各个类别的样本数。

Python Enhancement Proposal #8，简称PEP8

2.命名

PEP 8建议采用不同的方式来给Python代码中的各个部分命名，这样在阅读代码时，就可以根据这些名称看出它们在Python语言中的角色。

  • 函数、变量及属性用小写字母来拼写，各单词之间用下划线相连，例如：lowercase_underscore

  • 受保护的实例属性，用一个下划线开头，例如：

  • _leading_underscore

  • 私有的实例属性，用两个下划线开头，例如：__double_leading_underscore

  • 类（包括异常）命名时，每个单词的首字母均大写，例如：CapitalizedWord

  • 模块级别的常量，所有字母都大写，各单词之间用下划线相连，例如：ALL_CAPS

  • 类中的实例方法，应该把第一个参数命名为self，用来表示该对象本身

  • 类方法的第一个参数，应该命名为cls，用来表示这个类本身

单下划线和双下划线

python中通常用下划线作为前缀来表示属性的访问权限
具体来说，有两种下划线类型，单下划线 _ 和双下划线 __

  • 单下划线表示收保护的属性，可以被访问，但是不被认为是公共的API
  • 双下划线表示为”私有属性“，他们只能在累的内部被访问，无法从外部直接访问，尝试从外部时，会返回AttributeError异常

4.字符串的格式化

4.1 C风格的格式化

Python中常常使用%来格式化字符串

  • %左边的文本模板叫做格式字符串(format string)
  • %右边可以写多个值所构成的元组，用来替换格式字符串中的相关符号

C格式风格的格式化字符串

python中的格式说明符%s,%d等一般来自c语言

4.2 c格式的说明符有四个缺点

  • %右侧的元组中如果传参顺序发生了变化，就会出错
  • 填充模板之前，经常要先处理哪些要填进去的值，导致整个表达式过长
  • 如果想要让一个值填充字符串中的多个位置，就需要在%右侧多次重复该值
  • 把dict写到格式化表达式中会让代码变多

4.3 高级格式化.format()

  • 使用{}替代原来的占位符%s,%d等
  • 使用字符串.format(填充值1，填充值2)的格式来表达

你可以在{}里写个冒号，然后把格式说明符写在冒号的右边，用以规定format方法所接收的这个值应该按照怎样的格式来调整。在Python解释器里输入help('FORMATTING')，可以详细查看str.format使用的这套格式说明符所依据的规则。

但是这种方式并不能解决缺点2

21.1 什么是闭包

闭包是什么？以下是chatgpt给出的解释：

在 Python 中，闭包是指一个函数可以访问另一个函数内部的变量，并且可以在该函数调用结束后仍然保持对这些变量的访问。这种情况发生在内部函数引用了外部函数的变量或参数时。

通常来说，闭包就是外层函数内部的内层函数，使用了外层函数中的变量

21.2 python引用变量

python在引用变量时，解释器会按照下面的顺序对各个作用域(scope)进行查找

  • 当前函数的作用域
  • 外围作用域（包含当前函数的其他函数的作用域）
  • 全局作用域，即包含当前代码的那个模块所对应的作用域
  • 内置作用域，包含len和str函数的作用域

如果这些作用域内都没有定义相符名称的变量，就抛出NameError异常

26.修饰器

Python修饰器（Decorator）是一种特殊的函数，用于修改其他函数的功能，可以让函数在运行之前和执行完之后分别运行某些代码。它们的主要目的是使代码更加简洁，易于阅读和维护。

这是个很有用的机制，能够确保用户以正确的方式使用函数，也能够用来调试程序或实现函数注册功能，此外还有许多用途。
