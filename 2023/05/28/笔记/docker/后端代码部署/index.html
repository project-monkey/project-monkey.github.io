1.查询三个数据库容器的ip地址和端口号

注意，后端容器访问他们时，使用的是内部端口号

首先，使用docker ps查看容器列表，找到mongo,mysql和neo4j对应的容器id、容器名称

之后，使用命令docker inspect -f '{{range .NetworkSettings.Networks}}{{.IPAddress}}{{end}}' container_id_or_name查看容器对应的ip地址，之后将其写在config.py中

![image-20230528132855977](/Users/tianhaoran/Library/Application Support/typora-user-images/image-20230528132855977.png)

注意，后端容器访问他们时，使用的是内部端口号

2.修改配置文件

修改项目中的config.py

3.编写dockerfile

Dockerfile，位于项目的根目录enshi_smoke下

4.登录docker账号

docker官网：https://www.docker.com/

在官网注册账号，之后在终端登录

5.生成镜像

格式：docker build -t 账号名称/镜像名称 .

本项目：docker build -t tianhaoran/zssr_image .

![image-20230528134822307](/Users/tianhaoran/Library/Application Support/typora-user-images/image-20230528134822307.png)

6.推送和拉取镜像

以上的操作都是在本地进行的，下面要将在本地生成的镜像推送到服务器

首先，要在服务器也登录我们的docker，同4

之后，docker run -d -p 9223:8123 --name zssr_container tianhaoran/zssr_image:latest

7.尝试在本地访问

修改路由，位于vue.config.js，配置为服务器的ip地址+服务器的外部端口号

输入npm run install

进入localhost:9528，访问成功
