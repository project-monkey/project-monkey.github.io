2.命名

PEP 8建议采用不同的方式来给Python代码中的各个部分命名，这样在阅读代码时，就可以根据这些名称看出它们在Python语言中的角色。

  • 函数、变量及属性用小写字母来拼写，各单词之间用下划线相连，例如：lowercase_underscore

  • 受保护的实例属性，用一个下划线开头，例如：

  • _leading_underscore

  • 私有的实例属性，用两个下划线开头，例如：__double_leading_underscore

  • 类（包括异常）命名时，每个单词的首字母均大写，例如：CapitalizedWord

  • 模块级别的常量，所有字母都大写，各单词之间用下划线相连，例如：ALL_CAPS

  • 类中的实例方法，应该把第一个参数命名为self，用来表示该对象本身

  • 类方法的第一个参数，应该命名为cls，用来表示这个类本身

单下划线和双下划线

python中通常用下划线作为前缀来表示属性的访问权限
具体来说，有两种下划线类型，单下划线 _ 和双下划线 __

  • 单下划线表示收保护的属性，可以被访问，但是不被认为是公共的API
  • 双下划线表示为”私有属性“，他们只能在累的内部被访问，无法从外部直接访问，尝试从外部时，会返回AttributeError异常

4.字符串的格式化

4.1 C风格的格式化

Python中常常使用%来格式化字符串

  • %左边的文本模板叫做格式字符串(format string)
  • %右边可以写多个值所构成的元组，用来替换格式字符串中的相关符号

C格式风格的格式化字符串

python中的格式说明符%s,%d等一般来自c语言

4.2 c格式的说明符有四个缺点

  • %右侧的元组中如果传参顺序发生了变化，就会出错
  • 填充模板之前，经常要先处理哪些要填进去的值，导致整个表达式过长
  • 如果想要让一个值填充字符串中的多个位置，就需要在%右侧多次重复该值
  • 把dict写到格式化表达式中会让代码变多

4.3 高级格式化.format()

  • 使用{}替代原来的占位符%s,%d等
  • 使用字符串.format(填充值1，填充值2)的格式来表达

你可以在{}里写个冒号，然后把格式说明符写在冒号的右边，用以规定format方法所接收的这个值应该按照怎样的格式来调整。在Python解释器里输入help('FORMATTING')，可以详细查看str.format使用的这套格式说明符所依据的规则。

但是这种方式并不能解决缺点2

21.闭包

21.1 什么是闭包

闭包是什么？以下是chatgpt给出的解释：

在 Python 中，闭包是指一个函数可以访问另一个函数内部的变量，并且可以在该函数调用结束后仍然保持对这些变量的访问。这种情况发生在内部函数引用了外部函数的变量或参数时。

通常来说，闭包就是外层函数内部的内层函数，使用了外层函数中的变量

21.2 python引用变量

python在引用变量时，解释器会按照下面的顺序对各个作用域(scope)进行查找

  • 当前函数的作用域
  • 外围作用域（包含当前函数的其他函数的作用域）
  • 全局作用域，即包含当前代码的那个模块所对应的作用域
  • 内置作用域，包含len和str函数的作用域

如果这些作用域内都没有定义相符名称的变量，就抛出NameError异常

26.修饰器

Python修饰器（Decorator）是一种特殊的函数，用于修改其他函数的功能，可以让函数在运行之前和执行完之后分别运行某些代码。它们的主要目的是使代码更加简洁，易于阅读和维护。

这是个很有用的机制，能够确保用户以正确的方式使用函数，也能够用来调试程序或实现函数注册功能，此外还有许多用途。
